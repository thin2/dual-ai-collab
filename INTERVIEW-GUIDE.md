# 📋 Claude 需求访谈和任务规划指南

## 🎯 核心流程

当你需要开发新功能时，遵循以下流程：

```
用户提出需求
    ↓
Claude 深入访谈（使用 AskUserQuestion）
    ↓
生成详细的需求规范文档
    ↓
根据规范拆分任务
    ↓
写入任务板
    ↓
启动 Codex Worker
```

---

## 🔍 第 2 步：Claude 深入访谈

Claude 会使用 **AskUserQuestion** 工具对你进行多轮访谈，涵盖：

### 访谈维度

#### 1. **功能范围和目标**
- 这个功能的核心目标是什么？
- 主要用户是谁？
- 必须实现的功能有哪些？
- 可选的功能有哪些？

#### 2. **技术实现**
- 使用什么技术栈？
- 有没有技术限制或偏好？
- 需要集成哪些第三方服务？
- 性能要求是什么？

#### 3. **用户界面与体验**
- 用户如何与这个功能交互？
- 界面风格偏好？
- 响应式设计要求？
- 无障碍访问要求？

#### 4. **数据和安全**
- 需要存储什么数据？
- 数据安全要求？
- 隐私保护要求？
- 数据备份和恢复？

#### 5. **边界情况**
- 如何处理错误？
- 如何处理并发？
- 如何处理大量数据？
- 如何处理网络问题？

#### 6. **权衡取舍**
- 速度 vs 功能完整性？
- 简单 vs 灵活？
- 成本 vs 性能？
- 开发时间 vs 质量？

#### 7. **集成和依赖**
- 与现有系统如何集成？
- 依赖哪些其他模块？
- API 设计要求？
- 版本兼容性？

#### 8. **测试和验收**
- 如何验证功能正确？
- 测试覆盖率要求？
- 验收标准是什么？
- 如何进行用户测试？

## 📋 第 4 步：拆分任务并写入任务板

根据需求规范，Claude 会将功能拆分为具体的开发任务：

### 任务拆分原则

1. **粒度适中**：每个任务 1-3 小时完成
2. **独立可测**：任务之间尽量独立
3. **优先级明确**：P1 > P2 > P3
4. **验收标准清晰**：可测试、可验证

### 任务板示例

```markdown
# Codex 任务板

## 任务 #001: 创建数据库模型

**优先级**: P1
**状态**: OPEN
**分配给**: Codex
**创建时间**: 2026-03-06
**预计工时**: 2小时
**规范文档**: planning/specs/20260306-120000-用户认证系统.md

### 任务描述
根据需求规范第 3.4 节，创建用户认证相关的数据库模型。

### 技术要求
- 使用 SQLAlchemy ORM
- 创建 User 模型
- 创建 Token 模型
- 添加必要的索引

### 验收标准
- [ ] User 模型包含所有必需字段

- [ ] 数据库迁移文件创建
```

## 💡 访谈技巧

### Claude 提问原则

1. **非显而易见**：不问"需要数据库吗"这种显而易见的问题
2. **深入细节**：问"如何处理并发编辑冲突"而不是"需要并发吗"
3. **权衡取舍**：问"速度和功能完整性如何权衡"
4. **边界情况**：问"如何处理 10MB 的图片上传"
5. **用户体验**：问"用户如何发现编辑冲突"而不是"需要提示吗"

### 用户回答技巧

1. **具体明确**：说"响应时间 < 200ms"而不是"要快"
2. **提供上下文**：说明为什么这样选择
3. **说出优先级**：明确哪些是必须的，哪些是可选的
4. **提供示例**：如果有参考的产品，可以说明

## 🎯 总结

**完整流程**：

```
1. 提出需求
   ↓
2. Claude 深入访谈（5-10 轮问题）
   ↓
3. 生成需求规范文档（planning/specs/）
   ↓
4. 根据规范拆分任务
   ↓
5. 写入任务板（planning/codex-tasks.md）
   ↓
6. 启动 Codex Worker
   ↓
7. Codex 自动开发
   ↓
8. Claude 审计验收
```

**核心优势**：
- ✅ 需求明确，减少返工
- ✅ 规范文档可追溯
- ✅ 任务拆分合理
- ✅ 开发过程可控

现在你可以开始使用这个流程了！🚀
