### 2. 启动开发流程大幅优化 ✅

**新增内容**：

#### 启动前检查清单
```bash
⚠️  启动前检查：
```

#### 监控和管理命令
```bash
📊 查看实时日志
bash scripts/start-codex.sh -l
tail -f .dual-ai-collab/logs/worker.log

📋 查看任务进度
grep "状态:" planning/codex-tasks.md | grep -E "OPEN|IN_PROGRESS|DONE"

bash scripts/start-codex.sh -s

🔄 重启 Worker
bash scripts/start-codex.sh -s && bash scripts/start-codex.sh -t
```

### 4. 审计流程 ✅

**新增完整的审计流程**：

#### 触发词识别
- "审计 [功能名称]"
- "审查代码"
- "检查完成情况"

#### 审计步骤
1. 读取任务板（查看 DONE 任务）
2. 读取规范文档（了解验收标准）
3. 检查代码（功能、质量、安全、性能）
4. 更新任务状态（VERIFIED 或 REJECTED）
5. 生成审计报告

#### 审计评分标准
- 90-100：优秀
- 80-89：良好
- 70-79：及格
- 60-69：不及格
- 0-59：严重不合格

**解决问题**：完善工作流程，支持完整的开发-审计循环

### 5. 日志和监控 ✅

**新增详细的日志和监控指南**：

#### 日志级别
- ERROR：错误信息
- WARN：警告信息
- INFO：一般信息
- DEBUG：调试信息

#### 查看日志命令
```bash
# 实时查看
tail -f .dual-ai-collab/logs/worker.log

# 查看最近 100 行
tail -n 100 .dual-ai-collab/logs/worker.log

# 搜索错误
grep "ERROR" .dual-ai-collab/logs/worker.log

# 搜索特定任务
grep "任务 #001" .dual-ai-collab/logs/worker.log
```

#### 监控任务进度
```bash
# 统计任务状态
echo "OPEN: $(grep -c '状态: OPEN' planning/codex-tasks.md)"
echo "IN_PROGRESS: $(grep -c '状态: IN_PROGRESS' planning/codex-tasks.md)"
echo "DONE: $(grep -c '状态: DONE' planning/codex-tasks.md)"

# 查看当前执行的任务
grep -A 5 "状态: IN_PROGRESS" planning/codex-tasks.md
```

**解决问题**：审查报告问题 #3（日志机制不完善）

### 6. 故障排查指南 ✅

**新增 5 个常见问题的排查指南**：

1. **Codex Worker 无法启动**
   - 检查 codex 命令
   - 检查脚本权限
   - 检查日志文件

2. **任务一直处于 IN_PROGRESS**
   - 原因：Worker 进程崩溃
   - 解决：停止并重启 Worker

3. **任务执行失败**
   - 查看日志
   - 检查任务描述
   - 检查技术要求

4. **日志文件过大**
   - 自动轮转说明
   - 手动清理命令

5. **任务板格式错误**
   - 使用模板
   - 检查格式
   - 确保必填字段

**解决问题**：提高用户体验，减少使用障碍
