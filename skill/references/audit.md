# 审计规则与报告格式

## 审计触发条件

- 所有任务完成（状态为 DONE）
- 用户说出触发词："审计代码"、"审查任务"、"🔍 审计代码"

## 审计强度分级

不要默认所有任务都走完整三轮审计。先按复杂度和风险分级。

### 重审计

适用：

- P1 任务
- 跨模块改动
- 安全、权限、支付、数据迁移
- 新页面、核心用户路径、复杂交互

流程：

1. Spec Compliance Review
2. Code Quality Review
3. Claude 最终判定

### 标准审计

适用：

- 大多数 P2 任务
- 有一定实现复杂度，但风险可控

流程：

1. Spec Compliance Review
2. Claude 最终判定

如果第一轮暴露出明显质量风险，再补做 Code Quality Review。

### 轻审计

适用：

- 小型 P3 任务
- 小范围 bugfix
- 工具函数、脚本微调、样式细修

流程：

1. Claude 快速核验验收标准与变更范围
2. 必要时升级为标准审计

## 审计流程

如果当前环境启用了 superpowers，审计优先采用其方法论：

- 第一轮：`spec-reviewer` 风格，审查“是否按任务描述和验收标准完成”
- 第二轮：`code-quality-reviewer` 风格，审查“实现质量、维护性、边界与验证缺口”
- 修复阶段：`receiving-code-review` 风格，把问题整理为可执行修复清单

如果没有 superpowers 环境，则沿用下述同等结构的双阶段审查。

### 第一轮：规范符合性审计（Spec Compliance Review）

对每个 DONE 状态的任务，先检查“是不是完成了该做的事”，而不是先谈代码风格。

推荐 prompt 关注点：
```
你是规范符合性审查员，请审查以下任务的代码实现：

任务：[任务标题]
相关文件：[文件列表]
验收标准：[验收标准列表]

请检查：
1. 功能是否符合任务目标
2. 验收标准是否逐项被覆盖
3. 是否存在遗漏实现或明显偏题
4. 是否有需要人工确认的模糊点

输出格式：
- 评分（0-100）
- 结论（PASS / FAIL）
- Blocking Issues（逐条列出）
- Missing Coverage（逐条列出）
```

第一轮完成后：
1. 保存结果到 `planning/audit-reports/YYYYMMDD-任务XXX-spec-review.md`
2. 更新任务板 `**Codex审查**` 字段

### 第二轮：代码质量审计（Code Quality Review）

第二轮重点不再重复“有没有做”，而是判断“做得是否稳、是否易维护、是否还有明显回归风险”。

推荐关注点：

1. 代码结构是否清晰，是否易于维护
2. 是否引入边界遗漏或回归风险
3. 是否缺少关键验证或测试
4. 是否存在明显安全、性能、可靠性问题

第二轮完成后，进入 Claude 最终判定：

1. 读取任务板，查看哪些任务 DONE 且第一轮/第二轮结果已生成
2. 读取 spec review 和 code quality review 结果
3. 读取规范文档了解验收标准
4. 检查代码文件
5. 综合两轮审查意见，做出最终判定
6. 更新任务板 `**Claude审查**` 字段
7. 更新任务总状态（VERIFIED / REJECTED）和审计评分
8. 生成审计报告

### 第三轮：自动修复（REJECTED 任务）

Codex 读取审计报告进行修复。修复完成后：
1. 任务状态从 REJECTED 改回 DONE
2. 重置 Codex审查 和 Claude审查 为"未审查"
3. 重新进入第一轮审计

建议修复前先将问题整理成“问题 -> 影响 -> 修复动作 -> 验证方式”的清单，而不是把整份审计原样丢给执行器。

**修复循环上限**：每个任务最多修复 3 次。超过则标记 FAILED。

## 建议的默认策略

- P1 / 高风险任务：重审计
- P2 / 普通功能任务：标准审计
- P3 / 小修小补：轻审计

## 审计评分标准（0-100）

- 90-100：优秀，完全符合要求
- 80-89：良好，基本符合要求，有小问题
- 70-79：及格，符合基本要求，但需改进
- 60-69：不及格，存在明显问题
- 0-59：严重不合格，需要重做

## 审计报告格式

文件路径：`planning/audit-reports/YYYYMMDD-任务XXX-audit.md`

```markdown
# 审计报告：任务 #XXX

**审计时间**: YYYY-MM-DD HH:MM:SS
**任务标题**: [标题]
**修复轮次**: 第 X 轮（首次 / 修复后复审）

## 第一轮：规范符合性审查
- **评分**: XX/100
- **结论**: PASS / FAIL
- **发现问题**:
  - [问题 1]
  - [问题 2]

## 第二轮：代码质量审查
- **评分**: XX/100
- **结论**: PASS / FAIL
- **发现问题**:
  - [问题 1]
  - [问题 2]

## Claude 审查结果
- **评分**: XX/100
- **结论**: VERIFIED / REJECTED

## 代码质量
- [评价]

## 功能正确性
- [评价]

## 安全性
- [评价]

## 最终判定
- **综合评分**: XX/100（Codex 和 Claude 评分加权平均）
- **最终结论**: VERIFIED / REJECTED
- **改进建议**:
  - [建议]
```
