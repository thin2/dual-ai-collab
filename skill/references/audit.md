# 审计规则与报告格式

## 审计触发条件

- 所有任务完成（状态为 DONE）
- 用户说出触发词："审计代码"、"审查任务"、"🔍 审计代码"

## 审计流程

### 第一轮：Codex 审计

对每个 DONE 状态的任务，调用 Codex 进行代码审查。

Codex 审查 prompt 模板：
```
你是一个资深代码审查员，请审查以下任务的代码实现：

任务：[任务标题]
相关文件：[文件列表]
验收标准：[验收标准列表]

请检查：
1. 功能是否符合验收标准
2. 代码质量（可读性、可维护性）
3. 安全问题（注入、XSS、敏感信息泄露等）
4. 性能问题（内存泄漏、N+1 查询等）
5. 边界情况处理

输出格式：
- 评分（0-100）
- 结论（PASS / FAIL）
- 发现的问题（逐条列出）
- 改进建议（逐条列出）
```

Codex 审查完成后：
1. 保存结果到 `planning/audit-reports/YYYYMMDD-任务XXX-codex-review.md`
2. 更新任务板 `**Codex审查**` 字段

### 第二轮：Claude 审计

Claude 读取 Codex 审查结果，结合自己的代码审查，做出最终判定：

1. 读取任务板，查看哪些任务 DONE 且 Codex 已审查
2. 读取 Codex 审查报告
3. 读取规范文档了解验收标准
4. 检查代码文件
5. 综合 Codex 和 Claude 的审查意见，做出最终判定
6. 更新任务板 `**Claude审查**` 字段
7. 更新任务总状态（VERIFIED / REJECTED）和审计评分
8. 生成审计报告

### 第三轮：自动修复（REJECTED 任务）

Codex 读取审计报告进行修复。修复完成后：
1. 任务状态从 REJECTED 改回 DONE
2. 重置 Codex审查 和 Claude审查 为"未审查"
3. 重新进入第一轮审计

**修复循环上限**：每个任务最多修复 3 次。超过则标记 FAILED。

## 审计评分标准（0-100）

- 90-100：优秀，完全符合要求
- 80-89：良好，基本符合要求，有小问题
- 70-79：及格，符合基本要求，但需改进
- 60-69：不及格，存在明显问题
- 0-59：严重不合格，需要重做

## 审计报告格式

文件路径：`planning/audit-reports/YYYYMMDD-任务XXX-audit.md`

```markdown
# 审计报告：任务 #XXX

**审计时间**: YYYY-MM-DD HH:MM:SS
**任务标题**: [标题]
**修复轮次**: 第 X 轮（首次 / 修复后复审）

## Codex 审查结果
- **评分**: XX/100
- **结论**: PASS / FAIL
- **发现问题**:
  - [问题 1]
  - [问题 2]

## Claude 审查结果
- **评分**: XX/100
- **结论**: VERIFIED / REJECTED

## 代码质量
- [评价]

## 功能正确性
- [评价]

## 安全性
- [评价]

## 最终判定
- **综合评分**: XX/100（Codex 和 Claude 评分加权平均）
- **最终结论**: VERIFIED / REJECTED
- **改进建议**:
  - [建议]
```
